Este proceso acepta un archivo tar que ha sido comprimido con gzip. La extensión de archivo para este formato es ".tgz", pero también se suele usar la extensión ".tar.gz". La estructura del archivo comprimido debe seguir la siguiente nomenclatura: "hicn/beshare/icon.hvif". En este ejemplo, se proporciona un ícono vector HVIF para el paquete "beshare".

En ocasiones el ícono vector no está disponible. En este caso se puede entregar una combinación de imágenes PNG de 16px, 32px y 64px. Para esta situación utilice la siguiente nomenclatura en los archivos: "hicon/beshare/16.px".

Cada paquete que tenga los íconos modificados por este proceso pierde los los anteriores datos de ícono.

La importación de un archivo tar de íconos es "atómica", es decir, o se procesa el archivo por completo o no se procesa en absoluto. Esto está relacionado con errores de ejecución como datos inválidos que impiden que el proceso se ejecute en su totalidad.

El éxito en la operación así como los problemas serán reportados al usuario mediante la generación de un archivo CSV.
