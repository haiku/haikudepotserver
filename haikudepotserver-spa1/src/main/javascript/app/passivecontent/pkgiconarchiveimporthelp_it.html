Questo processo accetterà un file di archivio tar che è stato compresso con gzip. La tipica estensione di file per questo formato di archivio è ".tgz", ma a volte viene utilizzata anche un'estensione di file ".tar.gz". I file nell'archivio dovrebbero avere la seguente nomenclatura; "code>hicn/beshare/icon.hvif". In questo esempio, viene fornita un'icona vettoriale HVIF per il pacchetto "beshare".

A volte i dati delle icone vettoriali non sono disponibili. In questo caso può essere fornita una combinazione di immagini PNG quadrate da 16, 32 e 64 pixel. In questo caso, verrebbe utilizzata la seguente nomenclatura delle voci di archivio; "hicon/beshare/16.px".

Per ogni pacchetto con icone modificate da questo processo, i dati dell'icona precedente verranno rimossi.

L'importazione di un file tar di icone è "atomic" il che significa che elaborerà interamente l'archivio o non modificherà i dati lato server. Questo è in relazione a errori di runtime come dati di input non validi che impediscono del tutto l'esecuzione del processo.

Problemi come pacchetti mancanti e il corretto caricamento di un'icona vengono segnalati all'operatore tramite un file CSV generato.
